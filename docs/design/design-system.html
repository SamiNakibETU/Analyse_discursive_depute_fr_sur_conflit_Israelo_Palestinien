Niveaux de lecture

Toute figure doit être lisible à 3 distances. Ce n'est pas optionnel : c'est la définition d'une bonne dataviz éditoriale.

DistanceCe que l'œil perçoitÉlément responsable
Vue d'ensemble (2–3 m)La hiérarchie, la direction générale, le contrasteGrandes formes, couleur d'accent, composition SVG
Lecture (40–60 cm)Les catégories, les tendances, les titresTitres de planche, labels de blocs, lignes de force
Étude (15–25 cm)Les valeurs exactes, la méthode, les nuancesAnnotations Courier Prime, footer technique, ×, **, µ

07

Règles éditoriales

La rigueur graphique ne suffit pas. Le texte doit avoir la même autorité que les formes.

❌ À éviter
Distribution des cadres discursifs par bloc politique
Évolution temporelle du score de stance sur 26 mois
Comparaison des volumes de production entre groupes
Matrice des effets d'événements sur le positionnement
✓ À viser
La gauche parle souffrance. La droite parle sécurité.
Les positions sont figées. Seul le Centre hésite.
Groupe A monopolise : 4× plus de textes, 6× plus de posts
Le Centre réagit à tout. Le Groupe B ne bouge pas.
■Tout code court (G.RAD, E1, A, B…) est accompagné de son nom complet au moins une fois dans la figure.
■Les événements portent toujours : code + nom complet + date (ex : E1 · Ordonnance CIJ · 26 jan. 2024).
■Les données absentes sont signalées : × + "absent". Jamais un blanc vide non expliqué.
■L'absence est une information. Les cellules non-significatives restent visibles (hachures + valeur en gris).
■La direction analytique est toujours annotée en texte. La couleur seule ne suffit pas.
■La légende est intégrée au graphique. Pas de légende flottante séparée qui force un aller-retour oculaire.
■Le bandeau de codage (légende globale) est défini une seule fois en haut de publication. Pas répété.
■La description technique est une clé de lecture, pas une définition de variable.

Tokens d'implémentation

:root {

  --rule:   #D4D0CA;  /* filets et grille */
  --mute:   #888888;  /* annotations secondaires */
  --accent: #E8413C;  /* adapter par projet */
}
/* D3.js — objet C (Color + Config) */
const C = {
  ink:    '#111',
  bg:     '#fff',
  rule:   '#D4D0CA',

  accent: '#E8413C',    // adapter par projet

09

Checklist de livraison

À valider avant de considérer une planche terminée.

Le titre dit ce que la donnée prouve, pas ce qu'elle montre
La description technique donne une clé de lecture immédiate
Chaque code court est accompagné de son nom complet
Les données absentes sont signalées (×, hachures, gris)
La légende est intégrée au graphique
La direction analytique est annotée en texte, pas seulement en couleur
Les valeurs sont lisibles à 3 distances (vue, lecture, étude)
Pas de border-radius sur les barres
Pas de drop shadow ni d'animation
Pas plus de 3 épaisseurs de trait dans un même graphique
Fond blanc pur (#FFFFFF)
Une seule couleur d'accent (pas deux)
Pas d'arc-en-ciel de couleurs pour les catégories
Chaque chart utilise un vocabulaire visuel distinct
Les pictogrammes ont le même poids optique à 14px
Pas de fonte par défaut (Inter, Roboto, system-ui)
