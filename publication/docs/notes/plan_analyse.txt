### 2.3 Fix A1 - Circularité du lexical score
**Contexte** : La validation croisée lexical score ↔ stance_v3 était circulaire (le lexical score utilisait stance_v3 comme filtre d'ancrage).

**Correction** :
```
# ANCIEN (circulaire) : ancres = blocs filtrés par stance ≥ 1 ou ≤ -1
# NOUVEAU : ancres = blocs entiers (aucun filtre par stance)
ancres_pro_pal = v3[v3.bloc == "Gauche radicale"]["text_lemmatized"]
ancres_pro_isr = v3[v3.bloc == "Droite"]["text_lemmatized"]
```
- Recalculer le TF-IDF par député
- Cosine similarity de chaque député avec ancre pro-PAL vs ancre pro-ISR
- Lexical score = sim(pro_PAL) − sim(pro_ISR), normalisé [-1, +1]
- Corrélation avec stance_v3 moyen par député : Spearman ρ
- Comparer avec l'ancien ρ = 0.947 → si le nouveau ρ reste > 0.85, la validation tient

### 2.4 Fix B4 - Panel fixe pour la polarisation
**Contexte** : La polarisation lexicale croissante (τ = +0.376) peut être un artefact d'attrition.

**Correction** :
- Identifier les députés du "panel fixe" : ≥ 1 texte/mois pendant ≥ 18 mois de la période POST
- Recalculer la cosine distance inter-blocs mensuelle UNIQUEMENT sur ce panel
- Mann-Kendall τ sur la nouvelle série
- Comparer : τ_all_deputies vs τ_panel_fixe
- Si τ_panel ≈ 0 → la polarisation est un artefact de composition
- Si τ_panel reste > 0 → la polarisation est réelle

### 2.5 Cohérence interne v4

## CE QUI DISTINGUE CE PROJET

Pour un recruteur intelligence/défense, les points saillants sont :

1. **OSINT-anchored** : Chaque analyse est ancrée dans des événements réels et datés. Ce n'est pas du NLP dans le vide - c'est de l'analyse de l'impact discursif d'événements géopolitiques concrets.

2. **Multi-méthode** : LLM annotation (v3/v4) + NLP dictionnaire (FEEL, Fightin' Words) + statistiques (ITS, Mann-Kendall) + scaling non-supervisé (PCA/Wordfish). La triangulation des méthodes est LA marque d'un bon analyste.

3. **Honnêteté méthodologique** : Les corrections A1/B4 montrent que tu identifies et corriges les biais de ton propre pipeline. Les caveats sur Wordfish montrent que tu comprends les limites des outils.

4. **Variables originales** : `ceasefire_call`, `genocide_framing`, `icc_warrants_position`, `ceasefire_breach_blame` - ces variables n'existent dans aucun autre corpus. C'est du travail de conception d'instrument de mesure, pas juste de l'application de packages.
