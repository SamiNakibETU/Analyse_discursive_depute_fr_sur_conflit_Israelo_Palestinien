## 💡 Conseils d'Utilisation

### 1. Découpage par Lots
Perplexity fonctionne mieux avec des lots de **20-30 députés** :

```
Lot 1 : Députés 1-30
Lot 2 : Députés 31-60
Lot 3 : Députés 61-90
...
```

### 2. Priorisation
Commencer par les députés avec le plus d'interventions (voir Phase 1) :

```
Trouve les comptes Twitter des députés suivants (par ordre de priorité) :

HAUTE PRIORITÉ (10+ interventions) :
1. Jean-Noël Barrot (61 interventions) → ?
2. Gabriel Attal (24 interventions) → @GabrielAttal ✅
3. Élisa Martin (17 interventions) → ?
...
```

### 3. Vérification Croisée
Demander à Perplexity de vérifier les résultats :

```
Peux-tu vérifier que ces comptes Twitter sont corrects ?

- Élisa Martin → @ElisaMartinGre
- Jean-Noël Barrot → @jnbarrot
- Benjamin Haddad → @benjaminhaddad

Confirme que ce sont bien les députés français actuels.
```

## 🛠️ Après la Recherche Perplexity

### 1. Créer fichier JSON avec résultats

```json
{
  "Élisa Martin": "ElisaMartinGre",
  "Jean-Noël Barrot": "jnbarrot",
  "Sabrina Sebaihi": "SabrinaSebaihi",
  "Benjamin Haddad": "benjaminhaddad",
  ...
}
```

Sauvegarder dans : `config/twitter_handles_perplexity.json`

### 2. Fusionner avec mapping existant

```bash
cd final

# Fusionner les 2 fichiers JSON
python -c "
import json
existing = json.load(open('config/twitter_handles.json'))
perplexity = json.load(open('config/twitter_handles_perplexity.json'))
existing.update(perplexity)
json.dump(existing, open('config/twitter_handles.json', 'w'), indent=2, ensure_ascii=False)
print(f'Fusionné: {len(existing)} comptes')
"
```

### 3. Re-générer candidats (Phase 2)

```bash
# Relancer Phase 2 avec mapping enrichi
python scripts/phase2_generate_candidates.py
```

Maintenant il y aura **beaucoup moins** de députés à valider automatiquement.

### 4. Lancer Phase 3 sur les restants

```bash
# Valider uniquement ceux non trouvés par Perplexity
node scripts/phase3_validate_accounts.js
```

## 📈 Stratégie Optimale

### Approche Hybride (Recommandé)

1. ✅ **Perplexity** : Recherche manuelle des comptes évidents (~50-70%)
2. ✅ **Phase 3 automatique** : Validation patterns pour les restants (~20-30%)
3. ✅ **Recherche manuelle ciblée** : Google pour les cas difficiles (~10%)

### Estimation Temps

- **Perplexity** : 20-30 min (par lots)
- **Validation manuelle** : 10-15 min
- **Phase 3 automatique** : 30-45 min (sur restants seulement)

**Total : 60-90 min** au lieu de 2-3h avec Phase 3 complète
